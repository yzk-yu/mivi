## 一、核心亮点(先看这三个)

### 1. 多维度用户画像 —— 越聊越懂你

MIVI 不靠收集个人信息，而是从**每一句对话**里实时提取偏好信号，维护一套 **8 维偏好向量**：

| 维度 | 含义 | 维度 | 含义 |
|------|------|------|------|
| `pace` 节奏 | 想悠闲还是高效 | `aesthetic` 审美 | 是否在意环境/拍照 |
| `budget` 预算 | 价位敏感度 | `social` 社交 | 偏热闹还是安静 |
| `energy` 体力 | 能走多少路 | `adventure` 探索 | 爱小众还是经典 |
| `patience` 耐心 | 能不能接受排队 | `decision_load` 决策 | 想自己挑还是交给管家 |

这套画像驱动**所有推荐排序**：同样一句「找个咖啡馆」，审美高的人拿到的是景观店，社交低的人拿到的是安静店。左下角「了解你 X%」实时可见，画像随对话生长。**画像只有 8 个数字，不含任何身份信息，会话结束即清。**

### 2. 提前规划 —— 一句话出一整天

> 「周六和女朋友去苏州，喜欢拍照，预算 500」

一句话直接生成**带时间线、真实交通、预算的完整一日行程**：咖啡 → 景点 → 餐厅 → 娱乐，每站都标注停留时长、站间交通方式与耗时、人均花费。每个推荐都讲清「**为什么排在这**」，并给出匹配度评分与理由标签。还能指定时间（今天/明天/周末）和点名活动（看电影、密室、展览…）。

### 3. 中途调整 —— 玩的当下随时改

出方案后进入「事中陪伴」模式，今日行程钉在右侧面板：

- **主动盯**：按真实时间盯着进度，到点主动提醒，并附带真实天气 / 排队 / 预算检查。
- **随时改**：想取消某站、改时间、加一站，直接在面板上操作，**即时重排**（含重新计算路上交通时间）。
- **会兜底**：出发前自动检查天气，雨天把户外项目换成室内备选（隐形 Plan B）；超时了自动压缩后续、跳过低优先级项目。

## 二、基于 OpenClaw 框架开发

本作品基于开源智能体框架 **[OpenClaw](https://github.com/openclaw/openclaw)** 开发，使用版本 **2026.6.1**。

OpenClaw 的核心机制是 **Skill 插件体系**：每个 Skill 由一份 `SKILL.md`（声明触发条件、能力、所需环境）加实现代码组成，框架按对话意图自动调度。MIVI 把本地生活能力拆成 **4 个严格遵循 OpenClaw 插件规范的 Skill**：

| Skill | emoji | 职责 | 何时被调用 |
|-------|-------|------|-----------|
| `user-profiling` | 👤 | 用户画像（幕后服务） | 每次用户输入时提取偏好；其他 Skill 需要画像时被调 |
| `local-dining` | 🍜 | 餐饮推荐与打分 | 出方案时为每类候选打分排序 |
| `local-entertainment` | 🎭 | 娱乐推荐 + 天气自适应 | 出发前检查，按真实天气决定户外/室内 |
| `smart-itinerary` | 🗺️ | 全天动线编排 + 超时纠错 | 安排行程、行程中出现超时/变化时 |

> 每个 Skill 都包含完整的 `SKILL.md`（含 YAML frontmatter 与 `openclaw` metadata）和 `scripts/` 实现代码，位于 [`skills/`](./skills) 目录。其中 `user-profiling` 作为幕后服务标记了 `disable-model-invocation`，专门为其余三个 Skill 提供画像数据——**四个 Skill 协作**（画像喂进打分）是本作品的核心设计。
>
> Skill 在运行时被 MIVI 后端真实调用（通过 [`backend/skill_bridge.py`](./backend/skill_bridge.py) 桥接，每次调用都有 `[skill] ✓` 日志可查），并在调用失败时回退到后端等价逻辑，保证线上稳定。OpenClaw 框架本身加载这 4 个 Skill 的截图见 [`docs/`](./docs)。

详细的框架集成说明见 **[docs/OpenClaw集成说明.md](./docs/OpenClaw集成说明.md)**。

## 三、数据安全与合规

赛题明确要求「不得收集或存储任何真实用户个人信息」。MIVI 的设计：

- **不收集身份信息**：没有登录、没有账号、没有手机号。偏好只是 8 个 0~1 的数字，匿名 session 存内存，会话结束即清，不落库。
- **位置用完即焚**：定位仅来自「用户说的地名」或「用户显式授权的 GPS」；GPS 坐标只用于当次就近搜索，**用完即丢，绝不存储、绝不与身份绑定**，前端有明确标注。
- **偏好可模拟可显式输入**：用户可以直接打字描述偏好，也可以点选节奏等选项，完全符合「模拟数据或用户显式输入」的要求。

| 数据 | 来源 |
|------|------|
| POI / 路径 / 天气 / 地理编码 | 高德地图官方 API（真实） |
| 意图理解 / 推荐理由 / 评分细分 | DeepSeek 大模型（真实，可切换 Kimi/通义） |
| 电影演唱会、点评评价 | 模拟数据（无公开 API，合规标注，不爬取） |
| 用户偏好 | 对话输入 → 8 维向量，匿名内存态 |

## 四、怎么跑

### 在线体验

> **作品链接：http://122.51.240.252:5500/mivi.html**

直接用浏览器打开即可体验完整交互（推荐 Chrome / Edge 桌面端宽屏）。

体验路径建议：在对话框输入一句话（如「周六和女朋友去苏州，喜欢拍照，预算 500」）→ 选择城市与节奏 → 查看生成的一日行程 → 点开任一站看「为什么排在这」→ 在右侧面板调整时间或取消某站，观察即时重排。

> 说明：在线 Demo 经 HTTP 访问，浏览器「用我的位置（GPS）」在非 HTTPS 下会被禁用，体验时请用「快捷选城市」或直接打字告知城市；GPS 就近定位功能在本地 HTTPS 环境下完整可用。

### 本地运行

**前端**（零依赖，单文件）：用浏览器打开 `frontend/mivi.html` 即可。

**后端**（FastAPI）：
```bash
cd backend
pip install -r requirements.txt
cp .env.example .env          # 填入高德 / DeepSeek key（不填也能跑，自动走 Mock）
set -a; source .env; set +a
uvicorn main:app --reload --port 8080
```
交互式 API 文档：`http://localhost:8080/docs`

> 后端支持 Mock / 真实接口一键切换：`MIVI_USE_MOCK=true` 时全程使用合规演示数据、无需任何 key；为 `false` 时走真实接口，缺某项 key 时该项自动回退，不影响其余。

## 五、项目结构

```
mivi/
├── frontend/
│   └── mivi.html              # 单文件前端，对话式交互 + 今日行程面板
├── backend/                   # FastAPI 后端
│   ├── main.py                # API 接口（意图解析 / 出方案 / 重排 / 出发前检查 …）
│   ├── core.py                # 8 维偏好向量 + 增量纠错状态机
│   ├── plan_builder.py        # 真实出方案：搜 POI → 打分 → LLM 编排 → 补交通
│   ├── scorer.py              # 显式偏好匹配打分器
│   ├── activities.py          # 活动优先级表（看电影 / 密室 / 展览…）
│   ├── amap_client.py         # 高德客户端（POI / 路径 / 天气 / 地理编码）
│   ├── llm_client.py          # 可切换大模型层（DeepSeek / Kimi / 通义）
│   └── skill_bridge.py        # 桥接层：后端 ↔ OpenClaw Skill 真实调用
├── skills/                    # 4 个 OpenClaw Skill（SKILL.md + scripts/）
│   ├── user-profiling/        # 👤 用户画像（幕后服务）
│   ├── local-dining/          # 🍜 餐饮推荐
│   ├── local-entertainment/   # 🎭 娱乐 + 天气自适应
│   └── smart-itinerary/       # 🗺️ 行程编排 + 超时纠错
└── docs/
    └── OpenClaw集成说明.md     # 框架集成详细说明 + Skill 加载截图
```
